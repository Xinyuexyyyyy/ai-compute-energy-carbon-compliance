# NotebookLM 提示词

## 一、每次新会话先发送的总约束

```text
你是我的研究助教，只能根据当前Notebook中的来源回答，不要用常识补齐来源没有提供的事实。

回答时必须区分五类内容：
[政策事实] 政府或正式标准明确写出的内容；
[论文结论] 论文在特定方法、数据和假设下得到的结果；
[PPT主张] 王永真课件提出的概念、案例或建议；
[研究推断] 根据多项来源作出的解释；
[待核验] 当前来源不足、存在冲突或只有二手材料支持的说法。

每个关键判断都要注明来源名称，并尽可能给出PPT页码、政策章节、论文题目或DOI。不能把“鼓励探索”写成已经强制实施，不能把模型算例写成行业平均，不能把数据集词元交易等同于算力token定价或AI税。

当我的问题过大时，先指出其中混合了哪些不同层级，再把它收敛成一个可以学习、测量或研究的问题。不要给我机械的每日计划，也不要一次堆砌大量名词。先解释因果关系，再给下一步。
```

## 二、第一次导入后的来源审计

```text
请审计当前全部来源。按“政策原文/正式标准/论文或学术课件/研究整理/个人设想/低置信度线索”分类列出。说明每份来源能回答什么、不能回答什么、可能把我带偏的地方是什么。最后检查：当前来源是否足以支持“AI税即将出现”这一判断？如果不足，明确指出缺失的直接证据。
```

## 三、PPT精读

```text
请把王永真《数据中心节能降碳之算电协同》还原成一条完整论证链：为什么出现问题、提出了哪些技术路线、现实落地遇到什么障碍、作者最后提出什么制度和标准建议。每一部分列出对应页码。特别区分作者引用的政策、作者团队的模型结果和作者个人建议。
```

```text
只分析PPT第59至75页。解释TCO、算电协同激励系数、收益占电力成本比例、绿电直连成本和算冷全栈控制之间的关系。告诉我这些内容共同反驳了哪些过于乐观的商业假设。
```

```text
只分析PPT第79至87页。解释token计费、多维任务画像、用电画像、术语、标准和评价体系之间的先后关系。判断其中哪些已经有政策或标准承接，哪些仍是研究建议。
```

## 四、概念学习

```text
从一个实际的大模型推理请求开始，逐层向我解释：输入token、输出token、预填充、解码、FLOPS、GPU功率、利用率、运行时间、PUE、kWh、排放因子和成本是怎样连接起来的。每解释一层，给一个最小数字例子，并指出这一层最容易犯的错误。一次只讲两层，讲完后用一个问题检查我是否真正理解。
```

```text
比较PUE、CUE、WUE、SCI、每请求能耗、每千token能耗和单位算力碳排。不要只给定义，要说明各自适合回答什么决策问题、不能回答什么问题，以及它们能否相互替代。
```

```text
解释平均电网排放因子、边际排放因子、市场法绿电和物理绿电的区别。用“核算历史排放”“评价新增负荷影响”“安排低碳调度”“证明绿色采购”四个场景分别选择合适口径。
```

## 五、政策学习

```text
根据政策来源制作一条时间线。每项政策只提取：发布机关、文号或日期、约束对象、关键动词、量化目标、适用期限、与算电协同的关系。把“不得、应当、要求、支持、鼓励、探索”分开，不要将政策愿景写成已完成事实。
```

```text
检查以下推理是否成立：“国家提出词元交易，所以将按token征收AI税。”请分别从概念对象、政策文本、计量基础、征税法定原则和现实执行条件拆解。只使用来源能够支持的内容，缺乏法律来源的部分标为待核验。
```

## 六、前沿研究定位

```text
把当前来源中的研究分成：任务能耗测量、碳感知时序调度、跨区域任务迁移、需求响应、算冷协同、综合能源系统、市场与标准七类。对每类列出代表工作、数据、方法、主要结果、限制和成熟度。最后指出哪一类最适合能源专业本科生在三个月内形成可复现成果。
```

```text
比较王永真团队与Google碳感知计算、EuroSys 2024时空迁移限制研究、AI Energy Score和ML.ENERGY。重点回答：它们分别把系统边界画在哪里？哪些变量被控制？哪些变量被忽略？这些差异留下了什么可研究的问题？
```

## 七、研究问题收敛

```text
围绕“面向AI推理服务的任务级能耗、碳排与成本计量”提出三个本科生可完成的研究问题。每个问题必须给出：研究对象、因变量、自变量、数据来源、最小实验、评价指标、潜在贡献、最大风险和三个月内可交付结果。不要提出需要真实电网调度权、企业内部商业数据或大型GPU集群的问题。
```

```text
对“每千token能碳成本”做一次严格的可识别性分析。列出必须控制的变量、可观测变量、难以获得的变量、代理变量和误差来源。然后给出一个既可测量又不夸大普适性的指标定义。
```

```text
为研究题目设计最小实验。限定条件：2至3个开源模型、一种可获得GPU、三类任务、公开工具、无企业数据。输出实验矩阵、采集字段、重复次数、基线、计算公式、异常处理和结果图表。所有设计都要能对应到来源中的方法依据。
```

## 八、模型与工具学习

```text
请作为优化建模老师，带我从静态测算过渡到24小时任务调度。先定义集合、参数、决策变量、目标函数和约束，再解释每一项的现实含义。第一版只允许线性模型，并分别建立固定运行、最低电费、最低碳排和电碳联合四个场景。不要直接引入强化学习、鲁棒优化或复杂市场博弈。
```

```text
根据来源为我建立工具选择表：nvidia-smi/NVML、CodeCarbon、AI Energy Score、ML.ENERGY、Carbon Aware SDK、pandas、Pyomo、HiGHS、Gurobi、OpenDC。说明每个工具解决哪一步、输入输出是什么、学习成本、是否需要GPU或许可证，以及当前阶段是否必须。
```

## 九、研究设想写作

```text
根据当前来源起草一份4至6页的保研研究设想。结构为：问题背景、政策与研究依据、现有工作及不足、研究问题、计量边界、数据与方法、预期结果、创新性、局限和计划。全文使用审慎书面语言，不写“蓝海、风口、颠覆、必然”等商业词。所有数字必须附来源，不得出现无法验证的团队或市场信息。
```

```text
现在不要重写研究设想，只做审稿。以能源系统导师的视角找出：问题过大、概念混用、数据不可得、方法与问题不匹配、创新性虚假、结论先行和引用越界。按致命问题、重要问题、可修改问题排序，并给出最小修改方案。
```

## 十、口试与持续学习

```text
根据来源对我进行口试。一次只问一个问题，从概念解释逐步进入模型和政策。我的回答如果只是背定义，要继续追问“为什么”和“在哪些条件下不成立”。每五题总结一次我的知识缺口，并只指定下一块学习疆域，不给每日打卡计划。
```

```text
把我本次提问暴露出的知识缺口记录成三列：缺失概念、它阻碍了哪个研究判断、应回到哪份来源的哪一页或哪一节。只记录真正影响研究的问题，不扩展无关知识。
```

## 十一、商业材料使用时的额外约束

```text
当前Notebook含有早期商业规划和市场调研。它们只能用于提出假设，不能作为市场规模、客户需求、价格、人脉和竞争格局的最终证据。请建立“已证实/有信号/纯假设/已被反证”四列表，对每项商业判断标注依据。凡是来源中出现“likely、推断、预计、有人会需要、蓝海、无人做”等表达，默认降为待验证假设。
```
