# AI 算力能碳研究主线与概念边界

## 一、问题是怎样产生的

大模型训练和推理首先是计算任务，但计算必须落在芯片、服务器和数据中心上，最终表现为功率、电量、冷却负荷和碳排放。过去，软件业务与能源系统之间隔着多层基础设施，使用者通常只看到云服务价格。AI 负荷规模增长以后，电力供应、机房容量、冷却和区域能源条件开始反过来限制算力扩张。

因此，研究对象不应只写成“AI+能源”，而应具体表述为：

> AI 任务如何被转换成可测量、可比较、可调度的能源与碳排对象。

## 二、核心转换链

完整链条可以表示为：

`业务或任务 -> token与时延 -> 计算量FLOPS -> 服务器负载 -> IT功率 -> PUE -> 总功率 -> 用电量 -> 碳排放 -> 成本与价格`

这条链中任何一步边界不清，后续的成本核算和市场机制都会失真。

### 业务或任务

业务决定模型要完成什么工作，并规定准确率、质量、时延和可靠性。两个模型即使输出相同数量的 token，也可能完成不同难度的任务，不能直接比较。

### token 与 FLOPS

token 是模型处理文本的离散单位，FLOPS 是计算量或计算能力的表达。二者之间没有固定换算关系，模型结构、输入长度、预填充与解码阶段、批处理和推理框架都会改变结果。

### IT 功率、PUE 与用电量

服务器功率只是数据中心总用电的一部分。PUE 是数据中心总能耗与 IT 设备能耗之比，用于纳入制冷、供配电等基础设施开销。PUE 适合评价设施效率，但不能单独代表算力效率或碳效率。

### 电量、碳排与成本

用电量乘以排放因子可估算运行碳排，但平均排放因子、边际排放因子、市场化绿电和物理供电关系对应不同问题。电费、绿电成本、碳成本、建设成本和服务违约成本也不能合并成一个未经说明的“绿色成本”。

## 三、PPT 真正提出的问题

王永真 PPT 的贡献，是把数据中心节能从单体设备效率扩展到四个层次：

1. 设备节能：服务器、供配电、液冷和余热回收。
2. 园区协同：算力、电力、冷力、储能和热力联合优化。
3. 跨区域协同：任务根据时延、电价、碳强度和网络条件迁移。
4. 市场与标准：统一术语、任务画像、计量、评价和交易机制。

PPT 第79页提出绿色、灰色、黑色 token 计费，第80页提出从业务、token、FLOPS、PUE、kW 到 kWh 的多维画像。这是学术建议，不是现行税制或已实施的统一定价规则。

## 四、与原始想法的交汇

原始想法认为，AI 创造的价值最终会进入统计、金融、税务和法律体系。这个长期判断具有研究价值，但中间缺少一个前提：监管和交易必须先知道“对象是什么、边界在哪里、数量如何测”。

因此，当前最适合占据的位置不是直接预测 AI 税，而是研究计量接口：

> 面向 AI 推理服务的任务级能耗、碳排与成本计量。

计量体系形成以后，才可能向下游延伸：

- 统计：形成任务、设施和区域层面的指标口径。
- 金融：评价项目成本、现金流、绿色属性和风险。
- 税务：识别应税对象、计税依据和主体责任。
- 法律：处理数据权利、披露责任、算法责任和争议证据。

这些是后续接口，不是三个月内并行学习的四个主专业。

## 五、研究时必须保留的反证

1. 算电协同并不必然带来很高收益。PPT 情景中的激励系数小于0.2，相关收益约占年度电力成本20%，说明建设成本和业务约束可能压过电价收益。
2. 时空迁移并非越复杂越好。国际研究表明，现实条件下的减碳上限可能明显低于理想结果，简单策略有时已经获得大部分收益。
3. 每 token 能耗不是常数。任何脱离模型、硬件、任务、批量、时延和地区条件的固定数字都缺乏可比性。
4. 词元交易不等于能源定价。数据集价值体系中的 token 与算力能耗计量属于不同政策线索。
5. 模型算例不是工程实绩。论文中的成本、购电量或碳交易降幅只能在其假设和参数范围内解释。

## 六、当前推荐的研究题目

主标题可以收敛为：

> 面向 AI 推理服务的任务级能耗、碳排与成本计量研究

若希望更贴近王永真团队，可以写成：

> 面向算电协同的 AI 推理任务多维画像与能碳成本计量

研究的第一层是计量，第二层是经济性验证，第三层才是低碳调度。这样既保留 AI 原生问题，也不会一开始承诺完成重资产平台或完整市场设计。
