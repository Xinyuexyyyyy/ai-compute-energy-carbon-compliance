# 第一轮主链学习

## 本轮只回答一个问题

> 一个AI任务怎样一步步变成可计量的电量、碳排和成本；其中哪些步骤是物理换算，哪些只是需要数据和假设的模型？

本轮不学习电力市场出清算法，不展开环保税、资源税、碳税，也不讨论token货币化。它们都依赖前面这条计量链是否成立。

## 开始前：先写现在的直觉

不查资料，用三到五句话回答：

> 我现在认为，一个AI任务的耗电量是怎样算出来的？



再给下面三类连线做标记：`确定`、`大概知道`、`不知道`。

```text
任务 → token → FLOPs → 负载率 → IT功率 → 热负荷 → PUE → 总电量 → 碳排/成本
```

## 第一遍：沿12个节点建立最小解释

| 节点 | 仓库中的最小含义 | 这一节点必须解释的连接 | 我的解释或疑问 |
| --- | --- | --- | --- |
| [[业务]]与[[任务]] | 用户结果与可执行工作单元 | 为什么先确定任务边界，不能直接从token算价值 |  |
| [[token]] | 输入输出规模的一种表示 | 为什么token不能直接等于电量 |  |
| [[FLOPs（运算量）|FLOPs]]与[[FLOPS（运算性能）|FLOPS]] | 工作量与计算速度 | token到FLOPs为什么依赖模型和阶段 |  |
| [[任务灵活性]] | 可延迟、中断、迁移或降级程度 | 为什么“能测量”不等于“能调度” |  |
| [[服务器负载率]] | 资源实际使用程度 | 为什么功率不随负载严格线性变化 |  |
| [[IT功率]] | IT设备瞬时耗电速率 | 功率怎样积分为电量，测量边界在哪 |  |
| [[热负荷]] | 需要被冷却系统移除的热 | 为什么发热与温度会反过来影响计算 |  |
| [[PUE]] | 设施总能耗与IT能耗之比 | PUE能说明什么，不能说明什么 |  |
| [[电量]] | 功率对时间的积分 | IT电量与设施总电量怎样区分 |  |
| [[电网排放因子]] | 单位电量对应的温室气体排放 | 为什么平均因子和边际因子回答不同问题 |  |
| [[TCO]] | 生命周期总拥有成本 | 为什么节电不必然等于项目值得做 |  |
| [[任务画像]]与[[多维画像]] | 可比较、预测和调度的结构化字段 | 为什么画像是前面测量结果的汇总，不是凭空设计 |  |
