# 任务运行方式与调度空间

## 本轮主体关系与核心问题

关系：AI 服务 -> 任务运行方式 -> 调度空间。

核心问题：同一业务目标为什么会形成不同运行方式的任务，以及哪种任务可以参与算电协同调度。

## 用户的初始理解

运行方式可以先区分为在线任务和定时任务。

## 追问中暴露的混淆

- 定时任务是常见安排方式，但不等于全部批处理任务。
- 批处理任务的关键不是“固定在某个时刻执行”，而是在满足截止时间前具有时间灵活性。
- 低谷执行不是无条件成立，仍受交付时限、资源、数据、网络和服务约束限制。

## 共同形成的解释

在线任务通常要求用户等待期间立即获得结果，时延约束强，调度空间较小。批处理任务不要求即时返回；它可以固定定时执行，也可以在一个截止时间窗口内选择执行时段。

因此，AI 服务在任务拆分时识别出批处理或其他具时间灵活性的任务，就可能把任务安排到算力需求较低、电价较低或碳强度较低的时段执行。这里的核心不是“把算力调走”，而是改变任务实际执行的时间，必要时再讨论是否改变执行地点。

任务最晚何时完成首先由业务需求决定。业务主给出或确认交付截止时间，AI 服务再将其落实为任务的到达时间、截止时间和优先级。时间灵活性不是能源系统自行提供的条件，而是业务侧允许任务等待的空间。

在时间窗口内选择执行时段，至少要结合：任务截止时间与预计运行时长；当前 GPU 占用率和预测的可用容量；当前访问量及未来请求量预测；不同时段的电价；以及不同时段的碳强度。GPU 占用率描述当前状态，未来访问量和任务到达量帮助预测未来容量是否仍可用。

低电价时段不保证 GPU 容量充足。电价主要反映电力系统的供需和市场规则，而 GPU 容量取决于在线业务、既有预约和其他批处理任务；多个任务同时选择低价时段还会形成新的容量拥塞。

碳强度是某个地点、某个时段每使用一单位电量对应的碳排放量。它必须说明采用平均排放因子还是边际排放因子：前者描述该时段整体供电结构的平均结果，后者试图描述新增一单位负荷实际改变的边际电源。两者不能混用。

调度的第一优先级是满足任务的截止时间、服务约束和资源可用性，即先保证任务可行完成。只有在多个执行时段都可行时，才在成本、碳和其他目标之间比较取舍。

政府和标准机构设定法律、市场和披露规则；业务主或服务运营方在这些规则及合同约束下，为具体服务设定成本与碳的取舍。场景和时段改变的是电价、碳强度和容量等输入，不单独决定谁优先。

如果合同、政策或企业目标确认了碳排放上限，碳排放上限就是硬约束。调度不能为了降低成本而超过该上限；成本只能在按时完成任务且满足碳约束的候选方案中优化。

```text
交付时限
-> 在线或批处理等运行方式
-> 时间灵活性
-> 可否调整执行时段
-> 负荷、电价、碳强度与成本结果
```

## 最小反例

“用户在线等待生成结果”和“明早九点前生成日报”都可能来自报告类业务，但前者若延后到夜间低谷执行将违背服务约束；后者在截止时间前可能允许选择低谷时段。业务名称相同，不代表调度空间相同。

## 三类判断边界

| 类型 | 本轮内容 |
| --- | --- |
| 可以直接计算 | 已知运行时间和计量边界后，可以比较不同执行时段的电量、电价或给定碳强度。 |
| 必须实测或建模 | 批处理任务对不同时间段资源可用性、排队时延、功率、网络和冷却负荷的实际影响。 |
| 必须由业务、合同、制度或标准定义 | 截止时间、是否允许延后、SLA、优先级、数据可用窗口和调度授权。 |
