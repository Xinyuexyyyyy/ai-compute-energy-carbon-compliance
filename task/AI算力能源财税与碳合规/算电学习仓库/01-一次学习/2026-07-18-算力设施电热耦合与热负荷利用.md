# 算力设施电热耦合与热负荷利用

## 本轮对象关系与问题

本轮从 PPT 的三网协同关系进入主线，先建立以下对象网络：

```text
算力设施 ↔ 电力系统、绿电与储能
算力设施 ↔ 冷却、热力网络与余热利用
电力系统、绿电与储能 ↔ 冷却系统
冷却系统 ↔ 热力网络与余热利用
```

涉及三个尺度：

- 微观：芯片、服务器、机柜、供配电设备与冷却回路；
- 中观：数据中心、冷却系统、储能和算力综合能源系统；
- 宏观：电力系统、绿电、热力网络和余热用户。

关系同时包含电能转换、热量转移、储能的时间搬移，以及价格、监测数据和控制信号形成的反馈，不是一条唯一的线性顺序。

本轮继续追问：IT 设备形成的热负荷怎样被利用，热量又怎样被储存。

## 共同形成的关系解释

### 1. 数据中心具有两种能源身份

数据中心一方面是电力系统中的高可靠用电负荷，另一方面又是持续产生热量的热源。它内部的最小功率关系可以写为：

```text
设施总功率 = IT 功率 + 冷却功率 + 供配电及其他功率
设施总电量 = 设施总功率对时间的积分
```

近似稳态且边界明确时，IT 设备消耗的电功率最终大部分转化为热负荷。但温度变化还受芯片、冷板、冷却液、空气和建筑热容影响，不能把 IT 功率曲线直接当作温度曲线。

任务信息在这里是进入算力设施的局部接口，只提供服务质量、时间约束和可调窗口，不承担整个研究路线的全局主线。

### 2. 算力设施与电力系统的双向关系

电力系统向数据中心提供电能、绿电可用性、电价、碳强度和容量约束；数据中心向外返回负荷预测、实际功率曲线、运行状态和可调能力。

储能的主要作用是搬移能量的时间位置：电池搬移电能，蓄冷搬移冷却需求，蓄热搬移余热利用时间。储能提供技术灵活性，但不自动带来更低成本或更低碳排。

因此，算力设施的电力价值不仅取决于总电量，还取决于它在什么时间、什么地点以怎样的功率曲线用电，以及其中多少负荷能够在可靠性、温度和服务质量约束下安全调整。

### 3. 热负荷利用需要经过三层筛选

```text
产生的热量 → 可回收热量 → 可利用热量
```

- 产生的热量：芯片、服务器和机柜实际形成的热；
- 可回收热量：能够被空气或液冷回路、换热器带出的热；
- 可利用热量：温度、流量、时间、距离和用户需求都匹配的热。

因此：

```text
可利用热量 ≤ 可回收热量 ≤ 产生的热量
```

余热可以直接用于生活热水、园区供暖等低温需求，也可以经热泵升温后供给更高温热用户，或通过吸收式制冷等设备转化为冷量。热泵和制冷转换需要额外设备与电力，必须进入设施总功率和经济性核算。

### 4. 蓄热存的是介质中的热量

蓄热不是把热留在服务器或芯片中，而是通过换热器把热量转移到水、相变材料或地下储热介质中。

最常见的显热储存关系为：

```text
储存热量 = 介质质量 × 比热容 × 允许温差
```

例如，`50 m³` 水在允许温差为 `10℃` 时，粗略可以储存约 `0.58 MWh` 热量。这个数量只用于建立量级认识，实际工程还要扣除换热、管路、保温和运行损失。

运行逻辑为：

```text
热负荷高于当前热需求 → 给蓄热装置充热
热负荷低于当前热需求 → 释放已储热量
没有热需求或温度不满足 → 旁路散热
```

蓄冷与蓄热需要区分：蓄冷储存的是后续可用的冷量，主要用于转移制冷用电；蓄热储存的是已回收热量，主要用于匹配热用户需求。

### 5. 当前观测与判断边界

- 可直接计算：边界一致时，功率对时间积分得到电量；已知介质质量、比热和温差时，可估算显热储量。
- 必须实测或建模：供回水温度、流量、换热能力、实际可回收热量、热泵性能、储热损失和动态冷却响应。
- 必须由业务、合同或制度定义：热用户需求、供热温度、可靠性要求、结算价格、绿电和碳属性口径。
- 绿电属性、绿证和碳因子属于信息与核算关系，不是额外的物理电能流。
- 理论余热潜力不能直接写成可售热量、收益或减排量。
