# 空间迁移与数据中心差异

## 本轮主体关系与核心问题

关系：任务空间迁移 -> 数据中心物理条件 -> IT 功率、设施电量与碳成本。

核心问题：同一项任务换到不同数据中心执行时，为什么不能默认总电量不变。

## 用户的初始理解

如果任务换到另一个数据中心，原先可以先假设总电量不变，再比较两个地点的电网和成本差异。

## 追问中暴露的混淆

- 任务的业务目标或理论计算量相同，不等于实际 IT 电量相同。
- 数据中心的 GPU 型号、运行效率、冷却方式和冷却组织方式、规模、PUE 都会改变能耗映射。
- PUE 只能描述设施总电量相对 IT 电量的比例，不能替代 GPU 和服务器侧的功率、运行时长与效率分析。

## 共同形成的解释

空间迁移可以保持任务目标和理论计算量不变，但不同数据中心的硬件、软件栈、资源拥塞、冷却系统和供配电条件不同，会改变任务的实际运行时间、IT 功率和设施附加能耗。

可以先用以下分层关系理解：

```text
任务与模型
-> GPU/服务器型号、利用率与运行时间
-> IT 电量
-> PUE 与设施总电量
-> 当地时段电价、排放因子与碳成本
```

在同一任务下，至少要分别比较 IT 侧和设施侧，不能直接把总电量设为相同。只有在硬件、运行效率、冷却和设施边界足够接近，或模型明确作出该假设时，才可以把电量不变作为简化条件。

跨数据中心比较所需的字段可以分为调度前输入和执行后结果：

| 层级 | 调度前输入 | 执行后结果 |
| --- | --- | --- |
| 任务侧 | 任务目标、SLA、理论计算量、预计运行时间、所需 GPU 类型 | 实际运行时间、实际执行量、完成质量 |
| 设备侧 | GPU/服务器型号、额定与实测功率曲线、运行效率 | 实际利用率、IT 功率与 IT 电量 |
| 容量侧 | 总容量、当前占用率、可用容量、预计任务到达量 | 实际排队、资源占用和完成情况 |
| 设施侧 | PUE 口径、冷却和供配电条件 | 设施总电量与实际运行 PUE |
| 电网与政策侧 | 时段电价、平均或边际排放因子、碳价或内部碳价 | 物理碳排、电费、碳成本和综合成本 |

碳排放通常是由设施总电量和适用排放因子得到的结果；碳价是把碳排结果转成经济结果的参数。总装机量只描述理论容量，必须与当前占用率和实际可用容量结合。

调度前的运行时间和电量可以先用历史同类任务的平均值作为基线，再结合当前任务的模型、输入输出规模、批量、并发、精度、GPU 类型和数据中心条件进行修正。这个预计值需要带有假设或不确定性范围，执行后再用实际运行时间和电量回填校准。

本轮优先识别出两个高影响特征：模型决定计算图、参数规模、算子和理论工作量；并发决定同时到达的请求量、批处理机会、GPU 利用率、排队和资源竞争。两者会共同影响运行时间、功率曲线和单位任务电量，但作用机制不同。

## 最小算例

同一任务在两个数据中心理论计算量相同：

```text
数据中心 X：IT 电量 100 kWh，PUE 1.3，总电量 130 kWh
数据中心 Y：IT 电量 110 kWh，PUE 1.6，总电量 176 kWh
```

即使 X、Y 的电网排放因子和电价相同，Y 也可能因为硬件效率和设施条件不同而产生更高的总电量、成本和碳排。

## 三类判断边界

| 类型 | 本轮内容 |
| --- | --- |
| 可以直接计算 | 在 IT 电量和 PUE 已知且边界一致时，可计算设施总电量。 |
| 必须实测或建模 | 任务在不同 GPU、运行效率、冷却系统和数据中心规模下的实际运行时间、IT 功率和总电量。 |
| 必须由业务、合同、制度或标准定义 | SLA、数据迁移权限、计量边界、PUE 统计口径、碳排披露和成本结算口径。 |
