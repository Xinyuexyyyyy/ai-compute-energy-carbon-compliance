# 一次学习索引

一次学习保存第一次接触材料时真实形成的理解。它不追求正确、完整或好看，只要求区分四件事：材料原意、自己的解释、暂时相信的判断、尚未解决的困惑。

## Agent 职责与边界

- Agent 只负责本目录 `01-一次学习/` 的学习记录。
- 每轮只推进一条主体关系，且一次只追问一个问题。
- Agent 不得创建、修改、整理或重写 `02-二次学习/` 中的任何文件。
- 不得将聊天逐字搬入笔记，也不得替用户伪造复述。

## 一次学习收束条件

- 用户能够用自己的话复述本轮主体关系与核心问题；
- 至少留下一个用户原先不知道的连接，并记录追问中暴露的混淆；
- 共同记录解释、最小算例或反例、结论边界与未解问题；
- 每个关键判断区分：可以直接计算、必须实测或建模、必须由业务、合同、制度或标准定义；
- 记录本轮证据及可信度，或明确尚未核验的原因。

## 二次学习

一次学习完成后不会自动进入二次学习。是否启动、何时启动以及如何安排 `02-二次学习/`，完全由用户决定。日期可以调整，`01/02`只表示理解轮次，不表示必须当天完成。
