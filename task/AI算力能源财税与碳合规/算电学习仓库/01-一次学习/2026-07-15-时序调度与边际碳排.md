# 时序调度与边际碳排

## 本轮主体关系与核心问题

关系：任务时序迁移 -> 数据中心负荷曲线 -> 电网碳排结果。

核心问题：同一批计算任务移到另一个时段执行时，为什么碳排结果可能变化，以及应使用什么口径判断额外碳影响。

## 用户的初始理解

高峰期进入算力中心的电可能混入火电，低谷期可能不需要火电而由绿电满足，因此换时段会改变碳排。

## 追问中暴露的混淆

- 高峰或低谷不直接等于火电或纯绿电；供电结构受地点、季节、天气和电网运行状态影响。
- 低谷时段也可能主要由稳定的化石能源机组供电，夜间也可能缺少光伏。
- 边际影响不是“预测外的电量”才有；任何相对基线增加或减少的一单位用电都具有边际影响。

## 共同形成的解释

将批处理任务从时段 A 移到时段 B，通常先改变数据中心的 GPU 负荷、IT 功率和用电曲线，而不是消除任务的计算量。若两个时段的电价或电网供电结构不同，同一电量会对应不同成本和物理碳排结果。

平均供电结构描述某地点、某时段整体供电的平均构成，适合平均核算；边际电源描述新增一单位负荷时实际需要增加出力的电源，更直接反映一项新增或迁移任务的额外碳影响。

任务核算和调度效益评价必须分开。任务最终在 B 时段执行时，记录该任务实际排放的核算应以 B 时段为准；但评价“从 A 挪到 B 是否带来减排效益”时，A 是未迁移情形下的反事实基线，必须同时比较 A 的避免排放与 B 的新增排放。

反事实基线 A 必须取调度前原本计划执行的时段，而不是事后任意选择一个高碳时段。基线的计划时间、任务电量假设和排放因子口径都应在调度前记录，才能使减排效益可核算、可复查。

```text
任务从 A 时段移到 B 时段
-> A 的负荷减少，B 的负荷增加
-> 应比较 A 的避免边际排放与 B 的新增边际排放
-> 得到迁移带来的边际碳排变化
```

## 最小算例

若任务在两个时段电量相同且均为 `1 MWh`，迁移的边际碳排变化可近似写为：

```text
时序迁移碳变化 ≈ B 时段边际排放因子 - A 时段边际排放因子
```

这只是口径示意。实际结果还可能受任务运行效率、冷却负荷、储能、电网约束和大规模任务同时迁移的反馈影响。

## 三类判断边界

| 类型 | 本轮内容 |
| --- | --- |
| 可以直接计算 | 在电量和排放因子口径已给定时，可计算不同候选时段的碳排差异。 |
| 必须实测或建模 | 边际电源、边际排放因子、任务迁移后的实际电量变化和大规模调度反馈。 |
| 必须由业务、合同、制度或标准定义 | 使用平均还是边际口径，碳排用于内部优化、披露还是结算，以及可接受的调度授权。 |
