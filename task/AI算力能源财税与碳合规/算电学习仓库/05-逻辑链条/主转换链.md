# 主转换链

## 总链

```text
业务需求
→ 计算任务
→ token、时延与计算量
→ 服务器执行和负载
→ IT功率与热负荷
→ 冷却、PUE与设施总电量
→ 碳排、成本和环境属性
→ 运行决策与经济激励
→ 计量、评价、标准和交易
```

## 三类关系不能混淆

### 可以直接计算

- 功率对时间积分为[[电量]]；
- 在口径确定后，电量乘[[电网排放因子]]得到运行碳排；
- 在边界一致时，设施总能耗除以IT能耗得到[[PUE]]。

### 必须实测或建模

- token到FLOPs；
- 负载率到IT功率；
- IT功率到温度与冷却功率；
- 任务迁移带来的时延、网络能耗和服务损失。

### 必须由制度或合同确定

- 任务与算力产品如何计价；
- 绿电和绿证如何声明环境属性；
- 哪种排放口径用于披露或结算；
- 数据字段、验证责任、市场准入和收益分配。

## 这条链能支持什么

- 找出任务级能耗核算缺少的测点和参数；
- 判断碳感知调度的结果是否只是核算转移；
- 检查一个节能方案是否具有真实TCO收益；
- 为[[计量]]、画像、标准和交易建立先后关系。

它不能直接证明AI税、token统一价值或全国算电市场一定会出现。
