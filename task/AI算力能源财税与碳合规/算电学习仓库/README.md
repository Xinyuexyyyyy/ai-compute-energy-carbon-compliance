# 算电学习仓库

这个仓库用于研究“AI算力任务如何被转换成可测量、可预测、可调度、可评价和可定价的电力、热力与碳排对象”。

它首先是一套研究环境，其次才是一套笔记目录。学习由问题推动：材料帮助提出或回答问题，概念卡稳定对象与口径，逻辑链解释关系，证据区约束结论，模型与输出检验自己究竟懂没懂。笔记是这些活动留下的痕迹，不是学习本身。

## 本仓库的学习循环

```text
研究问题
   ↓
01-一次学习：接触材料，留下当前解释、原文证据和真实困惑
   ↓ 关闭材料，主动复述
02-二次学习：换来源、找冲突、纠错并重建自己的解释
   ↓
00-概念：稳定单个对象、边界和口径
05-逻辑链条：解释对象之间为什么相连、在何种条件下断裂
   ↓
08-模型与实验 / 10-输出成果：用复算、建模或写作检验理解
   ↓
新的错误、证据和问题返回前面的任意一层
```

这不是单向晋升流程。实验可以推翻概念，政策原文可以改变问题，二次学习也可以发现一次学习连研究对象都选错了。

一次学习不需要“写得像论文”。它只需要保留四个东西：材料讲了什么、我目前怎样解释、证据在哪里、我卡在哪里。

二次学习不是誊抄或润色。它必须回答四个问题：第一次理解错在哪里、另一来源是否支持、关系怎样成立、在哪些条件下不成立。完成后先尝试不看资料复述，再决定哪些内容值得进入概念卡或逻辑链。

逻辑链条只负责连接已经定义的概念。每个节点应使用Obsidian双链，每条连线要说明关系类型，例如“决定、换算为、约束、反馈影响、用于评价”，不能只画箭头。

## 从参考方法中只保留三件事

本仓库不继承外部“笔记驱动学习引擎”的课程结构、章节路由或固定工作流，只借鉴三个经过实践仍有价值的动作：

1. 新概念出现时，先检查自己缺少哪个前置概念；
2. 每轮学习结束前关闭资料，用自己的话复述；
3. 答错、困惑和被推翻的判断必须留痕，因为它们是下一轮学习的起点。

其余学习方法在[[01-学习方法实验|学习方法实验]]中按实际效果检验，不预先写成教条。

## 来源规则

每个重要判断尽量标记：

- `[政策事实]`：政府或正式标准原文明确写出；
- `[论文结论]`：在特定数据和假设下得到；
- `[PPT主张]`：课件提出的概念、案例或建议；
- `[研究推断]`：根据多个来源形成的解释；
- `[待核验]`：当前材料不足或存在冲突。

不得把“鼓励探索”改写成已经强制实施，不得把模型算例改写成行业平均，不得把数据集词元交易直接等同于算力token定价或AI税。
