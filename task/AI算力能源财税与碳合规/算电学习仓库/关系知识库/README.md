# 关系知识库

关系知识库是在一次学习笔记之上维护的“当前知识状态层”，不再建立一套平行学习笔记。现有 `00—11` 编号已经承担固定职责，因此本层使用不带编号的目录名，避免改动原有编号和导航。

## 与现有内容的区别

- 原始材料：PPT 图片、OCR、论文、政策和案例，保存材料本身及其出处。
- 概念卡：说明单个对象是什么、口径是什么、边界在哪里。
- 一次学习笔记：记录一次学习事件、用户理解变化和当时留下的问题，保留历史痕迹。
- 关系知识：维护当前有效的对象关系、作用方向、成立条件、边界和观测方式。
- 候选假设：保存 Agent 提出但尚未得到用户表达确认的新连接，不自动算作用户知识。
- 二次学习：是否进入、何时进入以及怎样整理完全由用户决定，本层不创建、修改或整理 `02-二次学习/`。

## 进入关系知识层的规则

```text
原始材料或一次学习笔记
→ 提取一条对象关系及其来源
→ 区分材料明示、合理推导、用户形成和 Agent 候选
→ 写明条件、边界、观测量与冲突
→ 更新关系的当前状态
```

一次学习笔记是事件记录，关系知识单元是当前状态。只有用户已经用自己的表达形成、修正、确认或应用的判断，才能标为 `user_formed`。Agent 讲解、摘要和推测只能保持为 `agent_candidate` 或其他符合来源的状态，不得因写入本层而自动升级。

当后续理解发生变化时，保留被修正判断的来源并标为 `superseded`，再记录新的当前判断；不回写或改造历史一次学习笔记。
