# 下一步 Task 计划

## 当前 Task 目标

先从研究对象地图、主转换链和反馈回路建立整体导航，再围绕研究对象关系进入一次学习。当前入口固定为：

```text
业务需求与验收目标
→ AI应用或服务工作流
→ 任务集合与最小任务画像
→ 计算资源与执行方式
```

本阶段不以逐题回答问题库为目标，而是先确认对象、关系、指标和下游接口。

## 一次学习闭环

1. 先确定完整研究路线中的当前位置、研究对象关系和本轮下游节点。
2. 从现有概念库调取本轮真正需要的 `2—5` 个概念卡；不重复讲解已经掌握的概念。
3. 用户先表达当前理解；Agent 在同一对象关系内一次只追问一个问题，但允许围绕该关系处理多个相关问题。
4. 只有出现关键混淆时才使用最小定义、算例或反例；具体数值、政策、标准、市场价格和正式研究结论才进入待核验范围。
5. 用户用自己的话复述理解变化，Agent 记录关系解释、相关概念、主链阻碍问题和局部暂缓问题。
6. 共同确认本轮打开的下游节点，并区分：可直接计算、必须实测或建模、必须由业务、合同、制度或标准定义。
7. 用户完成复述后，Agent 才将轻量结构化结果写入 `算电学习仓库/01-一次学习/`。
8. 下一轮入口由完整研究对象路线决定，不由最后一个局部未解问题自动决定；是否进入 `02-二次学习/` 完全由用户决定。

Agent 只负责 `01-一次学习/`，不得创建、修改、整理或重写 `02-二次学习/` 中的文件。

## 概念库调用规则

- 每轮先确定对象关系，再调取 `2—5` 个实际需要的现有概念卡。
- 不重复讲解用户已经掌握的概念；概念缺口只记录，不自动创建新卡。
- 一次学习笔记只链接本轮实际调用的概念卡。

## Task 阶段计划

### P0：整体研究对象导航

输入：[[主体—关系—问题总览（草案）]]、[[算电学习仓库/00-概念/00-研究对象/研究对象地图|研究对象地图]]、[[算电学习仓库/05-逻辑链条/主转换链|主转换链]]、[[算电学习仓库/05-逻辑链条/反馈回路|反馈回路]]。

完成标准：能够区分参与主体、物理研究对象、指标与结果、制度对象和反馈关系；历史笔记不再决定当前入口。

### P1：业务需求与任务画像

前提：业务需求是完整研究链的逻辑起点。本轮不讨论 GPU 功率、PUE、排放因子、内部碳价或 token 定价。

过程：围绕“业务需求与验收目标 → AI应用或服务工作流 → 任务集合与最小任务画像”推进。先调用概念卡，再记录用户理解；每次只追问一个问题；等待用户复述。

完成标准：形成业务目标、验收条件、服务工作流与任务画像之间的关系解释，并打开“任务画像 → 计算资源与执行方式”节点；不强行写成完整模型。

### P2：补齐最小证据链

只围绕当前一次学习暴露的具体数值、现行政策、标准条文、市场价格、真实工程效果或正式研究结论回看原始材料，并将“材料主张”“可验证证据”“待核验事实”分开。普通关系理解不自动进入证据审计。

完成标准：需要外部核验的判断有来源位置或待核验标记；不把模型结果、情景估算或建议写成现行事实。

### P3：按完整路线选择下一条关系或形成最小模型

根据完整研究对象路线和已打开的下游节点选择下一条关系，或确定是否已具备构建简化能碳成本模型的条件；不由局部暂缓问题单独改变路线。

完成标准：给出下一轮的单一对象关系、所需字段和不做的范围。

### P4：再连接商业机会验证

只有在研究主链形成稳定对象和指标口径后，才把结果映射到智算中心、液冷厂商、园区、能源企业或 AI 企业的具体场景。

完成标准：选择一个真实付费情境，定义可由测算、报告或访谈检验的假设；不将远端制度设想包装为现行需求。

## 一次学习笔记规则

新笔记采用轻量结构：

1. 本轮对象关系与问题；
2. 我的理解变化；
3. 共同形成的关系解释；
4. 相关概念、暂缓问题与下游节点。

“待核验事实”是可选部分；最小算例或反例只有在解除关键混淆时才使用。三类判断边界继续保留，但用正文中的简短标签或收束语表达，不机械填充固定章节。
