# 下一步 Task 计划

## 当前 Task 目标

从已初始化的概念与问题库中恢复可对话推进的研究节奏，先确定“研究对象之间的哪条关系最值得解决”，再进入证据和建模。

## Task 阶段计划

### P0：校正主体—关系—问题图

输入：[[主体—关系—问题总览（草案）]]、[[算电学习仓库/00-概念/00-研究对象/研究对象地图|研究对象地图]]。

完成标准：用户确认主体边界、关键关系和第一条要深入的关系；明确哪些关系只作背景，不在当前阶段展开。

### P1：完成第一轮互动学习

候选关系：AI 任务、模型 -> 服务器执行。

过程：先记录用户当前理解；再追问任务类型、模型、硬件、利用率与服务约束如何共同影响执行；最后区分可直接计算、必须实测或建模、必须由合同或制度定义的量。

完成标准：形成一个可讨论的任务到执行映射，列出关键量、假设和未解问题，不强行写成完整模型。

### P2：补齐最小证据链

只围绕 P1 暴露的关键缺口回看原始 PPT、政策原文或论文全文，并将“材料主张”“可验证证据”“待核验推测”分开。

完成标准：每个要保留的判断都有来源位置或明确的待核验标记；图表、公式和数值在视觉复核前不标为高置信。

### P3：选择第二条关系或形成最小模型

根据 P1 的结果，在“服务器执行 -> 机柜与冷却”和“数据中心 -> 绿电、储能与电网”之间选择下一条关系，或确定是否已具备构建简化能碳成本模型的条件。

完成标准：给出下一轮的单一研究问题、所需字段和不做的范围。

### P4：再连接商业机会验证

只有在研究主链形成稳定对象和指标口径后，才把结果映射到智算中心、液冷厂商、园区、能源企业或 AI 企业的具体场景。

完成标准：选择一个真实付费情境，定义可由测算、报告或访谈检验的假设；不将远端制度设想包装为现行需求。
