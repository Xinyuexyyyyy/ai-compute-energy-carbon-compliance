# 下一步 Task 计划

## 当前 Task 目标

以仓库现存 PPT 图片、OCR、逐页索引和 PPT 整理材料为首要路线依据，恢复算电热协同的整体研究图像。当前不把“业务需求 → AI 服务规格 → 工作流 → 任务集合”当成完整主线；它只作为业务对象进入计算系统的局部接口。

PPT 忠实的整体结构暂按以下网络理解：

```text
信息消费与 AI 业务需求
→ 异构业务任务与算力负荷
→ 算力设施与算力网络

算力设施与算力网络 ↔ 电力系统、绿电与储能
算力设施与算力网络 ↔ 冷却、热力网络与余热利用
电力系统、绿电与储能 ↔ 冷却系统
冷却系统 ↔ 热力网络与余热利用

上述关系共同影响：能效、服务质量、电量、碳排与成本
制度接口及反馈层：市场、标准、评价与监管
反馈信息：监测、价格、制度要求和运行状态
```

它包含微观设备、中观数据中心与园区、宏观算力网络与能源市场三个嵌套尺度，不是一条严格的线性课程链。

## 一次学习闭环

1. 先在 PPT 研究对象网络中定位当前位置、对象尺度和当前关系。
2. 按需调用已有概念卡，通常为 `2—5` 个，说明它们如何支撑当前关系；不足时不为凑数调用，也不自动创建新卡。
3. Agent 先提供候选结构、连接理由和边界，再让用户校正真正影响路线的分叉。
4. 同一时刻最多提出一个有实质意义的问题；一次学习不要求每轮都以问题结尾。
5. 如果用户回答已足以完成当前节点，Agent 应接受、总结、补充必要边界、更新对象关系并推进。
6. 只有具体数值、现行政策、标准条文、市场价格、真实工程效果或正式结论需要外部核验时，才增加待核验事实。
7. 当用户已经用自己的表达形成、修正、确认或应用了关键判断后，可以将轻量结果写入 `算电学习仓库/01-一次学习/`；不得仅为写笔记而强迫用户完整复述。
8. 下一轮入口由 PPT 研究对象路线决定，不由历史笔记的局部“下一轮入口”自动决定；不进入 `02-二次学习/`。

Agent 只负责 `01-一次学习/`，不得创建、修改、整理或重写 `02-二次学习/`。

## 研究对象导航

### 微观：设备与机架

芯片、服务器、机架、供配电、冷却回路、温度、功率和可靠性。

### 中观：数据中心与园区

数据中心、算力综合能源系统、储能、冷却、热力网络、余热利用和园区运营主体。

### 宏观：网络与市场

多数据中心、算力网络、电力系统、绿电与储能、算力市场、电力与碳市场、政府与标准机构。

### 关键关系

- 算力设施与算力网络 ↔ 电力系统、绿电与储能；
- 算力设施与算力网络 ↔ 冷却、热力网络与余热利用；
- 电力系统、绿电与储能 ↔ 冷却系统；
- 冷却系统 ↔ 热力网络与余热利用；
- 任务负荷灵活性 ↔ 电价、绿电、储能与电网状态；
- IT 负荷 ↔ 冷却 ↔ 温度 ↔ 服务质量与安全；
- 单体数据中心/园区 ↔ 多数据中心/区域网络；
- 技术效果 ↔ TCO ↔ 多主体激励；
- 监测与画像 ↔ 评价标准 ↔ 调度与交易；
- 市场、标准、评价与监管作为制度接口及反馈层作用于上述关系。

## Task 阶段计划

### P0：PPT 路线复位

先对照 PPT 图片、OCR、逐页索引、NotebookLM PPT 整理、研究对象地图、主转换链和反馈回路。完成标准：分开信息流、物理流、能量流、价值流和反馈流；标明微观、中观、宏观尺度；指出现有路线与 PPT 的偏差。

### P1：算力设施—电力系统—冷却/热力系统

围绕 PPT 的核心系统关系，识别算力设施、电力系统、冷却/热力系统和算力负荷分别是什么对象，以及电力、热、任务信息、价格和控制信号如何流动。本轮不进入 AI 服务规格、任务依赖、GPU 调度算法或具体优化公式。

完成标准：形成一张可复述的算—电—热对象关系图，并区分设备、园区和区域尺度。

### P2：任务负荷灵活性 ↔ 电价、绿电、储能与电网状态

在整体系统关系明确后，再讨论任务何时、何地、以何种服务约束参与调度；任务画像和 AI 服务规格只作为接口工具调用，不作为全局导航主线。

### P3：IT 负荷 ↔ 冷却 ↔ 温度 ↔ 服务质量

研究算力负荷如何进入功率、热、温度和可靠性反馈；区分平均能量核算与动态安全控制。

### P4：尺度扩展、经济与制度接口

再进入多数据中心、区域电网、TCO、多主体激励、评价、标准、交易和监管；不把远端制度设想写成现行事实。

1. 本轮对象关系与问题；
2. 我的理解变化；

4. 相关概念、暂缓问题与下游节点。

AI 服务规格、任务集合和任务画像只有在本轮确实作为对象接口使用时才记录，不得强行成为每轮模板。最小算例、反例和待核验事实均为可选项。
