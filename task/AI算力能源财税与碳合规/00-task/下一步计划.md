# 下一步 Task 计划

## 当前 Task 目标

从已初始化的概念与问题库中恢复可对话推进的研究节奏，先确定“研究对象之间的哪条关系最值得解决”，再进入证据和建模。

## 一次学习闭环

1. 选择一条主体关系，并提出本轮核心问题。
2. 用户先表达当前理解。
3. Agent 每次只追问一个问题；根据暴露的缺口给出反例、最小算例、必要概念或证据。
4. 用户用自己的话复述共同形成的解释。
5. 共同确认结论、适用边界与未解问题，并区分哪些量可以直接计算、哪些必须实测或建模、哪些必须由业务、合同、制度或标准定义。
6. 用户完成复述后，Agent 才将本轮的结构化结果写入 `算电学习仓库/01-一次学习/`。
7. 未解问题决定下一轮入口；是否进入 `02-二次学习/` 完全由用户决定。

Agent 只负责 `01-一次学习/`，不得创建、修改、整理或重写 `02-二次学习/` 中的文件。

## Task 阶段计划

### P0：校正主体—关系—问题图

输入：[[主体—关系—问题总览（草案）]]、[[算电学习仓库/00-概念/00-研究对象/研究对象地图|研究对象地图]]。

完成标准：用户确认主体边界、关键关系和第一条要深入的关系；明确哪些关系只作背景，不在当前阶段展开。

### P1：进入用户确认的第一条主体关系

前提：P0 已确认完整回路、当前研究起点和本轮要进入的主体关系。业务需求是完整研究链的逻辑起点，不重新争论这一点，也不预设立即进入“AI 任务、模型 -> 服务器执行”。

过程：严格执行“一次学习闭环”。先记录用户当前理解；每次只追问一个问题；根据缺口补充最小必要的反例、算例、概念或证据；等待用户完成自己的复述。

完成标准：用户完成复述，并共同形成一条可讨论的主体关系解释，列出关键量、假设、三类判断边界和未解问题；不强行写成完整模型。

### P2：为本轮暴露的缺口补齐最小证据链

只围绕 P1 暴露的关键缺口回看原始 PPT、政策原文或论文全文，并将“材料主张”“可验证证据”“待核验推测”分开。

完成标准：每个要保留的判断都有来源位置或明确的待核验标记；图表、公式和数值在视觉复核前不标为高置信。

### P3：选择下一条关系或判断是否需要最小模型

根据 P1 的未解问题选择下一条关系，或确定是否已具备构建简化能碳成本模型的条件；不预设固定顺序。

完成标准：给出下一轮的单一研究问题、所需字段和不做的范围。

### P4：再连接商业机会验证

只有在研究主链形成稳定对象和指标口径后，才把结果映射到智算中心、液冷厂商、园区、能源企业或 AI 企业的具体场景。

完成标准：选择一个真实付费情境，定义可由测算、报告或访谈检验的假设；不将远端制度设想包装为现行需求。

## 本次会话的启动点

先展示完整研究回路，分开说明参与主体、物理研究对象、指标与结果、反馈关系。随后只问一个问题：业务需求是谁的需求，以及希望通过 AI 应用或服务实现什么结果？
