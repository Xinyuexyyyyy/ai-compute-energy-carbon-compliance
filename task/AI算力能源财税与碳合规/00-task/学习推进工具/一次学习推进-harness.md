# 一次学习推进 Harness

## 运行前检查

- [ ] 当前路径仍在 `task/AI算力能源财税与碳合规/`。
- [ ] 当前对象关系能在主转换链或研究对象地图中定位。
- [ ] 已读取本轮需要的 `2—5` 个现有概念卡。
- [ ] 没有把历史笔记的“下一轮入口”当作强制路线。
- [ ] 没有进入 `02-二次学习/`。

## 候选结构检查

- [ ] 先说明上游输入、中间转换、下游输出。
- [ ] 明确参与主体、物理研究对象、指标/结果和制度对象的层次。
- [ ] 给出本轮服务规格或对象关系的候选结构。
- [ ] 标出哪些字段由业务主确认，哪些由 AI 服务整理，哪些要到资源阶段再确定。

## 提问闸门

只有满足以下至少一项才提问：

- [ ] 暴露两个研究对象之间的冲突或约束；
- [ ] 需要确定业务方、AI 服务方、算力方或制度方的责任；
- [ ] 答案会改变任务画像、任务集合、资源执行或调度路线；
- [ ] 需要比较两种有实质差异的解释；
- [ ] 用户当前理解存在会阻断主链的缺口。

若没有满足项：不提问，直接总结并推进。

## 推进检查

- [ ] 没有重复问“是否”“是不是”。
- [ ] 没有把宏观关系拆成无效术语定义题。
- [ ] 没有让用户替 Agent 生产基础框架。
- [ ] 用户回答已足够时，已接受、总结、补边界并更新对象关系。
- [ ] 已说明下一段主链接口，而不是被局部细节牵走。

## 收束检查

- [ ] 用户完成自己的复述后才写笔记。
- [ ] 笔记只记录本轮对象关系、理解变化、关系解释、概念和下游节点。
- [ ] 区分主链阻碍问题与局部暂缓问题。
- [ ] 只在确有需要时记录待核验事实、算例或反例。
- [ ] 三类判断边界已用简短标签或收束语表达。

## 停止条件

遇到以下情况立即停止扩展：

- 需要修改概念库；
- 需要进入 `02-二次学习/`；
- 需要把普通关系理解写成外部证据审计；
- 需要回答与当前对象关系无关的局部问题；
- 需要把尚未确认的政策、市场价格或工程效果写成事实。
