# 一次学习推进 Harness

## 路线来源检查

- [ ] 已优先读取相关 PPT 图片、OCR 或页索引。
- [ ] 已区分 PPT 明确关系、合理推导、仓库后续整理和 Agent 新增结构。
- [ ] 没有把主转换链或历史笔记反过来当成 PPT 原文。
- [ ] 没有把缺失的 PPT 页内容猜测成事实。

## 运行前检查

- [ ] 当前路径仍在 `task/AI算力能源财税与碳合规/`。
- [ ] 当前关系能在 PPT 对象网络中定位。
- [ ] 已标明微观、中观或宏观尺度。
- [ ] 已标明主转换、反馈或制度接口。
- [ ] 已读取本轮需要的 `2—5` 个现有概念卡。
- [ ] 没有进入 `02-二次学习/`。

## 候选结构检查

- [ ] 已区分参与主体、物理对象、指标/结果和制度对象。
- [ ] 已说明任务信息、电力、热、价格、监测数据或控制信号如何流动。
- [ ] 已说明上游对象、中间关系、下游对象和观测量。
- [ ] 没有预设每轮都经过 AI 服务规格、工作流或任务集合。

## 提问闸门

只有满足以下至少一项才提问：

- [ ] 改变两个 PPT 核心对象之间的关系；
- [ ] 改变设备、园区或区域尺度；
- [ ] 改变物理、业务、经济或制度边界；
- [ ] 改变关键约束或观测方式；
- [ ] 改变下一段主线路线；
- [ ] 用户理解存在会阻断当前对象关系的缺口。

若没有满足项：不提问，直接总结并推进。

## 推进检查

- [ ] 没有连续确认术语、任务名称或已经明确的共识。
- [ ] 没有让用户替 Agent 生产基础对象图。
- [ ] 没有被任务依赖、容量预留、内部碳价等局部细节牵走。
- [ ] 用户回答足够时，已接受、总结、补边界并更新关系。
- [ ] 下一节点来自 PPT 对象网络，而不是历史笔记的最后一个问题。

## 收束检查

- [ ] 用户完成自己的复述后才写笔记。
- [ ] 笔记只记录对象关系、理解变化、关系解释、概念和下游节点。
- [ ] 区分主链阻碍问题与局部暂缓问题。
- [ ] 只在必要时记录待核验事实、算例或反例。
- [ ] 没有修改或进入 `02-二次学习/`。

## 停止条件

- 需要新增或重建概念库；
- 需要把 AI 服务规格或任务集合重新升级为全局主线；
- 需要把普通理解写成外部证据审计；
- 需要把 PPT 建议、情景估算或模型结果写成现行事实；
- 需要在当前对象关系未收束时跳到无关细节。
